# 第3节 方法

本文面向黄龙病（HLB）场景下的动态生存预测任务。给定果园层面的共享环境时间序列以及树级静态特征，目标是在某一 landmark 时点上，基于截至该时点的历史前缀信息，预测该树在后续离散时期内发生终点事件的风险分布。围绕这一目标，本文的方法沿着 `period_ms -> period_ms_tree_query` 的路径逐步演化：前者首先将日级环境历史压缩为时期级多尺度表示，并在时期层面完成动态风险建模；后者进一步引入树特异的历史时期查询机制，使不同静态属性的树能够对同一段共享环境历史形成不同的关注模式。

## 3.1 问题定义与 Landmark Prefix 建模

### 3.1.1 动态 HLB 生存预测任务定义

设第 \(i\) 棵树的静态特征为 \(\mathbf{s}_i \in \mathbb{R}^{d_s}\)，对应果园在整个观测区间上的日级环境序列为 \(\mathbf{E}_{1:T}=\{\mathbf{e}_1,\mathbf{e}_2,\ldots,\mathbf{e}_T\}\)，其中 \(\mathbf{e}_t \in \mathbb{R}^{d_e}\)。为适配 HLB 的周期性测量流程，我们依据相邻测量节点将完整时间轴离散为 \(K\) 个时期，记为 \(\mathcal{P}_1,\mathcal{P}_2,\ldots,\mathcal{P}_K\)。对第 \(i\) 棵树，记其终点事件或删失对应的时期为 \(\tau_i \in \{1,\ldots,K\}\)，事件指示变量为 \(\delta_i \in \{0,1\}\)。

在任一 landmark 时期 \(l\) 上，模型只能访问该树在 \(l\) 之前的历史前缀信息，并输出 landmark 之后的条件离散风险分布：

\[
p_i^{(l)}(k)=\Pr(T_i=k \mid T_i>l,\mathcal{H}_i^{(l)}), \quad k=l+1,\ldots,K,
\]

其中 \(\mathcal{H}_i^{(l)}\) 表示截至 landmark \(l\) 的历史前缀。与此同时，模型还需给出尾部生存概率

\[
q_i^{(l)}=\Pr(T_i>K \mid T_i>l,\mathcal{H}_i^{(l)}).
\]

该定义包含两个关键特点。首先，环境历史主要来自果园层面的共享观测，因此所有树在时间维度上共享同一组环境变化背景。其次，风险评估对象是具体单株树木，不同树的静态属性、病程状态与结构位置不同，决定了它们不应对共享历史做完全一致的解释。后者正是后续引入树特异查询机制的直接动机。

### 3.1.2 Landmark Prefix Expansion 样本构造

为了将原始的单次随访记录转化为动态预测问题，本文采用 landmark prefix expansion 策略构造训练样本。对于第 \(i\) 棵树，只要其在 landmark \(l\) 时仍处于风险集中，即满足 \(l < \tau_i\)，就生成一个以 \(\mathcal{H}_i^{(l)}\) 为输入的 prefix 样本。于是，同一棵树会在多个 landmark 上贡献多个训练样本，而这些样本共享同一终点事件或删失结果，但对应不同长度的历史前缀。

这一构造带来两方面收益。其一，它显式模拟了临床或田间监测中的真实使用方式，即每到一个新的观察节点，都基于已有历史重新评估未来风险。其二，它显著扩大了可用训练样本数，使模型能够在多个历史长度上学习“历史前缀如何影响未来风险”的动态规律，而不必为每个 landmark 单独训练一套模型。

### 3.1.3 输入表示与时期离散化

在每个 prefix 样本中，输入由两部分组成：一部分是截至 landmark 的历史环境前缀，另一部分是树级静态特征。考虑到日级环境观测存在噪声大、时间粒度过细、直接建模难以稳定的问题，本文先根据测量节点将日级时间轴划分为时期，再在时期层面构造环境表示。这样，每个 prefix 样本可写为

\[
\mathcal{H}_i^{(l)}=\{\mathbf{z}_1,\mathbf{z}_2,\ldots,\mathbf{z}_l,\mathbf{s}_i\},
\]

其中 \(\mathbf{z}_k\) 表示第 \(k\) 个时期的环境特征向量。此处的 \(\mathbf{z}_k\) 只承担输入表示的角色，后续再由模型学习其低维 token 表征与风险含义。通过这一处理，原始的“长日序列 + 单次结局”问题被统一改写为“时期级历史前缀 + 树级静态特征 \(\rightarrow\) 未来离散风险分布”的动态生存建模问题。

## 3.2 `period_ms`：时期级多尺度动态生存建模

`period_ms` 的核心目标，是在不依赖日级 RNN 顺序编码的前提下，直接在时期层面完成动态风险建模。它首先将每个时期映射为一个具有多尺度统计含义的环境 token，再通过相对滞后感知的时期注意力，从 landmark 之前的历史中选出与当前预测最相关的时期信息。

### 3.2.1 时期级多尺度环境特征构建

对每个时期 \(\mathcal{P}_k\)，本文不简单使用该时期内环境变量的均值，而是构造一组多尺度统计量以形成时期特征向量 \(\mathbf{z}_k\)。具体而言，\(\mathbf{z}_k\) 由时期内环境变量的均值、标准差、极小值、极大值、变化斜率以及时期结束时刻的末端状态共同组成。这样做的原因在于，同样的时期均值可能对应完全不同的波动结构、变化趋势和末端暴露水平，而这些差异往往会影响疾病进展。

因此，\(\mathbf{z}_k\) 并不是对日级历史的简单压缩，而是一个兼顾强度、波动、极端性和阶段末状态的时期级摘要。它为后续模型在时期层面识别“关键暴露阶段”提供了更稳定的输入基础。

### 3.2.2 基于 KAN 的时期 Token 编码

得到时期特征向量后，`period_ms` 使用 KAN 投影器将每个 \(\mathbf{z}_k\) 映射到低维 token 空间：

\[
\mathbf{h}_k=f_{\text{KAN}}(\mathbf{z}_k).
\]

KAN 的优势在于，它通过一组可学习的一维非线性函数完成特征映射，能够更灵活地刻画环境统计量与风险相关模式之间的非线性关系，同时保留较好的结构可解释性。经过该步骤后，原始的时期特征被转化为一组语义更紧凑的时期 token \(\{\mathbf{h}_1,\ldots,\mathbf{h}_K\}\)，用于后续的历史选择与风险聚合。

### 3.2.3 相对滞后感知的时期注意力聚合

在 landmark \(l\) 上，模型仅在已观测时期 \(k \le l\) 上计算注意力权重。为刻画“距离当前 landmark 有多远”的相对时间关系，本文为每个历史时期引入相对滞后嵌入 \(\mathbf{r}_{l-k+1}\)，并基于时期 token 与滞后嵌入计算注意力得分：

\[
a_{l,k}=\mathbf{w}^{\top}\tanh(\mathbf{h}_k+\mathbf{r}_{l-k+1}), \quad k \le l.
\]

随后，在有效历史时期集合上进行 masked softmax，得到时期注意力权重

\[
\alpha_{l,k}=\frac{\exp(a_{l,k})}{\sum_{j \le l}\exp(a_{l,j})}.
\]

这一步的含义是：模型并不假设越近的时期一定越重要，而是让相对时间距离与时期内容共同决定历史贡献。由于 `period_ms` 的注意力打分仅依赖共享的时期历史与当前 landmark，而不依赖树级静态特征，因此当多棵树共享同一段环境历史时，它们在时期选择上倾向于形成相同或近似相同的注意力模式。这一性质为后续 `period_ms_tree_query` 的增强提供了清晰的切入点。

### 3.2.4 风险表征融合与离散时间输出

在得到时期注意力后，模型首先计算注意力上下文

\[
\mathbf{c}_l=\sum_{k \le l}\alpha_{l,k}\mathbf{h}_k,
\]

并对历史时期 token 做最大池化，得到另一种历史摘要

\[
\mathbf{m}_l=\max_{k \le l}\mathbf{h}_k.
\]

与此同时，树级静态特征 \(\mathbf{s}_i\) 被映射为静态摘要 \(\mathbf{u}_i\)。三者拼接后形成风险表征

\[
\mathbf{g}_i^{(l)}=[\mathbf{c}_l;\mathbf{m}_l;\mathbf{u}_i].
\]

随后，模型通过生存风险头输出 landmark 之后各时期的事件 logits 以及一个尾部生存 logit。为了确保风险只分配给未来时期，所有不晚于 landmark 的时期都会被显式 mask，最终通过 softmax 得到条件离散风险分布 \(\{p_i^{(l)}(k)\}_{k=l+1}^K\) 及尾部生存概率 \(q_i^{(l)}\)。至此，`period_ms` 在时期级表示空间内完成了从历史前缀到未来离散风险分布的映射。

## 3.3 `period_ms_tree_query`：树特异的历史时期查询增强

虽然 `period_ms` 已经能够在时期层面选择关键历史，但其时期选择机制仍主要由共享环境历史驱动，对不同树缺少显式条件化。因此，当多棵树面对相同的环境背景时，模型难以表达“不同树应如何以不同方式解释同一段历史”这一需求。`period_ms_tree_query` 正是在 `period_ms` 基础上，对历史时期查询机制进行树级静态条件化增强。

### 3.3.1 树级静态摘要驱动的 Query 构造

在 `period_ms_tree_query` 中，树级静态特征首先被映射为静态摘要向量

\[
\mathbf{u}_i=g_{\text{static}}(\mathbf{s}_i).
\]

该向量不直接替代环境历史，而是作为查询条件参与“如何检索历史时期”的决策。换言之，时期 token 仍然承担历史内容本身的表达，而树级静态摘要则回答“对于第 \(i\) 棵树，哪些历史时期更值得关注”这一问题。由此，模型形成了“query 来自树，key/value 来自时期历史”的基本结构。

### 3.3.2 静态条件调制的时期 Token 重标定

为了让静态条件不仅影响打分层，还能进入时期表征本身，本文对时期 token 引入轻量的 FiLM 式调制。具体地，静态摘要 \(\mathbf{u}_i\) 被映射为一组缩放项与平移项，从而对原始时期 token \(\mathbf{h}_k\) 进行重标定：

\[
\tilde{\mathbf{h}}_{i,k}=\mathbf{h}_k \odot \left(1+\phi_{\gamma}(\mathbf{u}_i)\right)+\phi_{\beta}(\mathbf{u}_i).
\]

这里的调制是软性的，其目标不是将静态特征机械复制到每个时期，而是让同一时期 token 在不同树条件下呈现不同的语义侧重。进一步地，模型还基于静态摘要学习一个门控向量 \(\mathbf{g}_i\)，并构造交互项

\[
\mathbf{m}_{i,k}=\tilde{\mathbf{h}}_{i,k}\odot \mathbf{g}_i,
\]

用于显式表征“某一树级属性与某一历史时期内容之间的耦合强度”。这一设计使静态信息能够以更细粒度的方式影响历史选择。

### 3.3.3 树条件加性注意力打分

在完成 token 重标定后，`period_ms_tree_query` 采用树条件加性注意力来计算历史时期得分。对 landmark \(l\) 下的历史时期 \(k \le l\)，其打分函数写为

\[
e_{i,l,k}=\mathbf{v}^{\top}\tanh\!\left(
W_p\tilde{\mathbf{h}}_{i,k}
+W_r\mathbf{r}_{l-k+1}
+W_s\mathbf{u}_i
+W_m\mathbf{m}_{i,k}
\right).
\]

随后同样只在已观测时期上做 masked softmax，得到树特异的时期注意力权重

\[
\alpha_{i,l,k}=\frac{\exp(e_{i,l,k})}{\sum_{j \le l}\exp(e_{i,l,j})}.
\]

与 `period_ms` 相比，区别不在于标签形式、样本构造或损失函数，而在于时期注意力的打分函数已显式依赖树级静态条件。因此，即使多棵树共享完全相同的环境历史，只要其静态属性不同，它们也可以学出不同的历史关注模式，这正是 `tree_query` 中 “tree-specific” 的来源。

### 3.3.4 树特异风险融合与可解释注意力输出

在树条件注意力下，模型得到树特异的历史上下文

\[
\mathbf{c}_{i,l}=\sum_{k \le l}\alpha_{i,l,k}\tilde{\mathbf{h}}_{i,k},
\]

并对调制后的历史时期 token 做最大池化，得到

\[
\mathbf{m}_{i,l}^{\max}=\max_{k \le l}\tilde{\mathbf{h}}_{i,k}.
\]

随后结合该历史最大池化摘要、静态摘要以及当前 landmark 的位置嵌入，共同形成增强后的风险表征：

\[
\mathbf{g}_{i,l}^{\text{tree}}=[\mathbf{c}_{i,l};\mathbf{m}_{i,l}^{\max};\mathbf{u}_i;\mathbf{e}_l].
\]

在此基础上，模型仍沿用 `period_ms` 的离散时间风险输出方式，输出未来各时期的条件事件概率与尾部生存概率。换言之，`period_ms_tree_query` 的核心增强并不在于重写输出空间，而在于让“谁来查询历史、如何选择历史”这一步变成树级条件化的。

同时，为了支持后续解释分析，模型保留时期级注意力权重，并可将其再映射回日级时间轴。具体做法是将某一时期的注意力质量均匀分配到该时期内部的有效观测日上，从而得到可视化友好的日级注意力轨迹。这样，模型不仅能够输出风险，还能够给出“哪些历史时期或历史日期驱动了当前风险判断”的解释线索。

## 3.4 多任务训练目标与正则化

对主线模型而言，训练目标由离散时间生存主损失、landmark 内排序损失、CT-delta 辅助监督以及注意力扩散正则共同组成。需要强调的是，本文的方法节仅保留当前主模型真实使用的目标函数，而不引入 legacy 日级环境重建辅助项，以避免叙事焦点偏离 `period_ms_tree_query` 本身。

### 3.4.1 条件离散时间负对数似然

对事件样本（\(\delta_i=1\)），模型直接最大化真实事件时期的条件概率；其负对数似然为

\[
\mathcal{L}_{\text{nll}}^{\text{event}}=-\log p_i^{(l)}(\tau_i).
\]

对删失样本（\(\delta_i=0\)），模型优化“在删失时点之后仍未发生事件”的概率，即

\[
\mathcal{L}_{\text{nll}}^{\text{censor}}
=-\log\!\left(
\sum_{k=\tau_i+1}^{K} p_i^{(l)}(k)+q_i^{(l)}
\right).
\]

该主损失与离散时间生存建模完全一致，直接约束模型在 landmark 条件下输出合理的未来风险分布。

### 3.4.2 Landmark 内排序损失

仅依赖似然项可能不足以充分拉开高风险与低风险样本的排序差异，因此本文进一步在相同 landmark 内构造可比较样本对。对在 landmark \(l\) 后较早发生事件的样本 \(i\) 及其可比较样本 \(j\)，定义从 landmark 到真实事件时期的累计风险为

\[
R_i^{(l)}=\sum_{k=l+1}^{\tau_i} p_i^{(l)}(k).
\]

随后采用 Dynamic-DeepHit 风格的排序约束，鼓励更早发生事件的样本具有更高累计风险。该项本质上服务于当前的 prefix 风险建模任务，其作用是提升模型对个体相对风险顺序的区分能力，而不是改变主标签定义。

### 3.4.3 CT-delta 辅助监督

在存在相邻测量值的 prefix 样本上，本文引入 CT 变化量预测作为辅助任务。记 landmark 相邻两次测量之间的 CT 变化量为 \(\Delta c_i^{(l)}\)，模型输出对应预测 \(\widehat{\Delta c}_i^{(l)}\)。实践中可先对目标做标准化，再在有效样本子集上计算 Huber 或均方误差损失：

\[
\mathcal{L}_{\text{ct}}=\ell\!\left(\widehat{\Delta c}_i^{(l)},\Delta c_i^{(l)}\right).
\]

这一辅助监督并不改变生存终点的定义，而是通过引入疾病进展强度信号，促使风险表征更敏感地捕捉病情变化方向与幅度。

### 3.4.4 注意力扩散正则

如果时期注意力过度塌缩到单一时期，模型虽然可能在局部训练样本上拟合得更快，但往往会削弱时期级历史选择的稳定性。为此，本文对时期注意力分布引入熵下界约束。记

\[
\mathcal{H}(\boldsymbol{\alpha}_{i,l})=-\sum_{k \le l}\alpha_{i,l,k}\log \alpha_{i,l,k},
\]

则扩散正则可写为

\[
\mathcal{L}_{\text{spread}}
=\max\!\left(0,\eta \log L_l-\mathcal{H}(\boldsymbol{\alpha}_{i,l})\right),
\]

其中 \(L_l\) 表示 landmark \(l\) 下的有效历史长度，\(\eta\) 为控制目标熵比例的系数。该正则的目的不是强迫注意力趋于完全均匀，而是防止注意力在存在足够历史信息时退化为极端单点选择。

### 3.4.5 总体优化目标

综合以上各项，模型的总体训练目标为

\[
\mathcal{L}
=\alpha \mathcal{L}_{\text{nll}}
+\beta \mathcal{L}_{\text{rank}}
+\gamma \mathcal{L}_{\text{ct}}
+\lambda \mathcal{L}_{\text{spread}},
\]

其中 \(\alpha,\beta,\gamma,\lambda\) 分别控制主似然项、排序项、CT 辅助项与注意力扩散项的相对权重。该目标函数保持了生存预测作为主任务的核心地位，同时利用排序监督、疾病进展辅助信号与注意力稳定约束，共同提升 landmark 条件下的动态风险建模能力。
